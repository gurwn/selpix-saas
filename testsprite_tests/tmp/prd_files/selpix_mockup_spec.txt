📋 1. 마진 계산기 검증 (/margin)
가장 로직이 중요하고 민감한 부분이므로 꼼꼼한 테스트가 필요합니다.

 기본 계산 검증: 정상적인 값을 입력했을 때 순수익과 마진율이 엑셀 계산 결과와 일치하는지 확인
 수수료율 적용 확인: 로켓배송(10.8%), 윙배송(6.5%), 위탁(8.0%) 선택 시 각각 다른 수수료가 적용되는지
 극한값(Edge Case) 테스트:
도매가가 0원이거나 마이너스일 때 처리
판매 배율이 1.0배(원가) 이하일 때 마이너스 마진 경고 표시 여부
 입력값 유효성 검사: 문자를 입력하거나 너무 큰 숫자를 넣었을 때 UI가 깨지지 않는지
📋 2. 쿠팡 상품 등록 API 검증 (/api/coupang/register)
백엔드 로직 중 가장 복잡한 부분입니다.

 필수 파라미터 누락 테스트: productName, price 등을 빼고 요청했을 때 적절한 400 Bad Request 에러를 뱉는지
 HTML 태그 정제(Sanitization): 상품 상세 설명에 <script> 같은 위험한 태그를 넣었을 때 잘 제거되는지
 전화번호 포맷팅: 010-1234-5678을 넣었을 때 +821012345678로 잘 변환되는지
 옵션 생성 로직: 옵션 값이 없을 때 단품으로 잘 생성되는지
📋 3. 대시보드 및 공통 UI (/)
사용자가 가장 먼저 마주하는 화면의 안정성을 점검합니다.

 API 실패 시나리오: json-server가 응답하지 않을 때 화면이 하얗게 변하지 않고 "데이터 로드 실패" 메시지가 뜨는지
 로딩 상태 확인: 데이터를 불러오는 동안 스피너(Loading Spinner)가 정상적으로 표시되는지
 반응형 동작: 창 크기를 모바일 사이즈로 줄였을 때 사이드바나 그래프가 깨지지 않는지
